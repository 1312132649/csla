RockfordLhotka replied on Wednesday, April 20, 2011

That may work fine, with the following limitations (off the top of my head):

  • You'll need to make sure your IEnumerable<T> is serializable
  • If you want to support a Silverlight or WP7 or Android or iOS client, you'll need to manually implement IMobileObject to make the list serializable (good luck, that's hard)
  • You won't be able to bind to all UI technologies, as they don't all bind against IEnumerable
  • Even when binding does work (such as in WPF/SL/WP7), you probably won't get automatic client-side sorting (though your mileage may vary)
  • Although the data portal will work against any concrete type, you still need to provide a criteria and implement the DataPortal_Fetch or factory Fetch method - that's a requirement for any type you retrieve via the data portal

My recommendation is that you look at the current CSLA 4 code techniques for using ROLB and ROB (as shown in the Using CSLA 4 ebooks). The amount of code required is quite low - and (assuming you want the features of the data portal and data binding) you'll almost certainly end up writing at least that much code to roll your own solution.
