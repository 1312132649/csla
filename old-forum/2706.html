Hi glenntoy,

thanks, I have the basic idea as you detailed it, but I was hoping something along the lines of this might be out there already (I'm slowly putting this together so it's in a very draft state right now:

Creating a new CSLA project:
  1. In Visual Studio, select New | Project and then select Other Project Types and Visual Studio Solutions
  2. Enter Name of project (eg Membership) and click on OK
  3. Right click on the Solution in the Explorer and choose Add | New project
  4. Choose Visual Basic (or Visual C#) and choose Class Library
  5. Enter the name of the library (eg Membership.Library) and click on OK
  6. From the File menu choose Add | New project
  7. Choose Visual Basic (or Visual C#) and choose Windows Application
  8. Enter the name (eg Membership.Windows) and click on OK
  9. We might need to add a Data project as well?
  10. Create a database.vb (see file in ProjectTracker.library)
  11. Make sure the References include csla.dll, system, system.configuration, system.data, system.transactions, system.xml in the <projectname>.Library project.
  12. Ensure the references in the .Windows project include csla.dll, enterpriseserviceshostvb, <projectname>.library.dll, system, system.configuration, system.data, system.transactions, system.xml
