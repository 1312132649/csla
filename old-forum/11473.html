RockfordLhotka replied on Sunday, July 15, 2012

The installer puts the files in

c:\program files {x86)\Marimer LLC\Csla\<version>

So it shouldn't mess up older versions, because I doubt you'd create a folder structure like that one :)

The current installer also puts the source and samples into zip files in that folder, along with binaries to support Silverlight 4 in a zip file.

The 4.5 installer is a little different, because Microsoft dropped the setup project type in VS12. But it basically does the same thing, and into the same folder structure.
