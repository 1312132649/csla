JoeFallon1 replied on Wednesday, October 04, 2006

You can't bind an NVL to a cbo in CSLA 1.x.

Unless you implement BindableList code supplied by one of the members of this forum.

It was written a few years back but is widely used.

Just search for it and you should find the Region of code you need to add to the NVL base class.

Then you can bind easily.

 

I recommend you use CSLA 2.1 for new projects though.

And I agree - if you need 1.x use the latest version so the pain of upgrading later will be lessened.

The key thing is that BrokenRules.Assert is discontinued.

You need to write rules the way that 2.x uses them and 1.53 is the closest thing to it.\
