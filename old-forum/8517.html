RockfordLhotka replied on Thursday, February 11, 2010

Generally it should work pretty well.

Known issues:

  • There are some breaking changes in MVC 2 (compared to MVC 1) that block the Csla.Web.Mvc code from building.
  • There are some issues using the Csla.Xaml PropertyStatus and BusyStatus controls in the VS10 designer (in 3.8 and 4) that I have on the to-do list to fix in CSLA 4.
  • We had encountered some issues with a couple of the Windows Forms controls in the .NET 4 client profile, and I think those are fixed in the current CSLA 4 bits.

Fwiw, I don't plan to address these issues in 3.8.x - they'll be addressed in 4.
