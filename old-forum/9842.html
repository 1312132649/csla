tiago posted on Tuesday, December 07, 2010

Since CTP1 there were 53 work items closed (28 features, 24 issues and 1 task). During this 60 days a lot of work has been done on several areas.

First the stereotypes:

  • EditableRoot is OK
  • EditableChild is OK
  • EditableRootCollection is OK
  • EditableChildCollection is OK
  • ReadOnlyObject is OK
  • ReadOnlyCollection is OK
  • NamevalueList is OK

The remaining 3 stereotypes

  • EditableSwitchable
  • DynamicEditableRoot
  • DynamicEditableRootCollection

will receive the same "love & care" that was given to the other stereotypes on CTP3. The same CTP3 will most likely see the birth of VB templates as well as other features.

The following features were greatly improved:

  • LoadingScheme/LazyLoad
  • Object Relations Builder
  • templates
  • validation both on ORB and on templates

Some useful UI improvements

  • hide useless properties on several PropertyGrid
  • drop down lists added everywhere they can be used
  • all drop down lists pre-select the current value(s)
  • in the Project panel, move up/down more than one Csla Object at a time

Last but not least, among the 24 issues solved are the Vista/Windows 7 usability issues.
