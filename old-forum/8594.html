I put a beta release of CSLA .NET 3.8.3 online - for Windows and Silverlight.

This is primarily a bug fix release, and should be quite stable.

The only non-bug change is an enhancement to the Silverlight data portal, allowing the creation of completely custom proxy/host pairs. It is now possible to create a completely different data portal channel that is not necessarily based off WcfProxy.

This change was made primarily to support an upcoming set of custom channels that use a commercial WCF extension, but it is a good change overall, since it also enables the use of TCP sockets or other communication technologies that might exist for Silverlight.
