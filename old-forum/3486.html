renegrin posted on Wednesday, September 05, 2007

I have a problem with CSLA smartdate implementation.

  • My Window Form UI as two textboxes, one for date and one for time each bound to the single datetime column from my datasource

I need only the date to display in my date textbox and only the time to display in my time textbox. Since I'm using devexpress dateedit and timeedit controls, the correct values are displayed when the user edits within the textboxe, but when the focus is shifted away, the default CSLA ISO date format is shown in both textboxes. How do I adjust this behavior?

 

renegrin replied on Thursday, September 06, 2007

Solved. Using my Devexpress timeedit control I was able to set the Properties.Mask.UseMaskAsDisplayFormat property to true. This displayed my values as they are shown in the editing mode. No CSLA required.
