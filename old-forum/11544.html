JonnyBee replied on Thursday, August 23, 2012

Search the samples download for UpdateChild and UpdateChildren.

Basically, you will hve

  • a list with child objects.
  • each child has Child_Insert/Child_Update/Child_Delete methods for update to database (like DataAdapter)
  • if "root" list you call the Save method - if child list (owned by another Parent object)  you typically call UpdateCild or UpdateChildren from within the parents DataPortal_XYZ or Child_XYZ methods.

ProjectTracker sample use this technique to update both Project and it's list of Resources within one transaction.
