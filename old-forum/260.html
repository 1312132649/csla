I have a windows form bound to a BO with child objects. It is possible that the child objects will have no records in the database. In this case I am having trouble binding a member of the child object to a control.

In the editable root I get the child like:

mChild = ChildBO.GetChild(ID, True)

If there are no corresponding database records I get a child object with string fields that are Nothing. The problem comes when I try to bind the fields in the ChildBO to controls.

Any suggestions on the best way to handle this?
