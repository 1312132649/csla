I am new to CSLA. Please allow me this question: What is a business object? What this root and child stuff all about?

An object is an object graph. It starts with a root object or a root list. This object can have children which could be a single object or a list. Child objects can have child objects (grand child objects) which are single objects or list. Is this correct?

JonnyBee replied on Monday, November 12, 2012

That's correct.

Just like a XML document -  you always have "root" object on top and it typically only gives meaning to Save the entire structure  - hence you can only call Save on the "root" object - not on Child objects.
