Hi,

I'm using SSRS client side reports for my applications reporting.  I have a feature that exports the report to PDF without using the ReportViewer control first by using the Render method of the LocalReport.

This works wonderfully, unless the resulting PDF is printed to a PostScript printer.  Such a printer prints a few lines, then spits out the page, and prints an error code on the next page.

We have some HP laser printers here, and if I install the PCL drivers, everything is fine.  If I uninstall them and install the PS printer drivers, I can reproduce the error.

I haven't been able to find any information on this; does anybody know of a solution? 

Thanks
Andy

I've narrowed it down a bit.  I have a single image, an PNG which is embedded into the LocalReport.

It seems that this image is causing the problem.  Why I have no idea...

ajj3085 replied on Thursday, August 30, 2007

I don't know if it was the particular PNG I was using, or if SSRS can't handle PNGs at all, but saving the image as a jpg and using that in the report has fixed the issue.
