RockfordLhotka replied on Thursday, May 22, 2008

That exception occurs because the data portal is set to use custom authentication and is expecting that the client has set up a principal object that is a BusinessPrincipal. This is a pretty common CSLA configuration issue, and so might not be related to your use of 2008.
