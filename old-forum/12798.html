ajj3085 replied on Tuesday, November 11, 2014

Since VPNs are usually slower than LAN access, I wouldn't recommend using the local data portal in that scenario as database updates are normally pretty chatty.  As for WCF, you'd configure it like you normally would, I would probably still use WCF over HTTPS for the clients connecting to a server behind the VPN.  Also using a WCF remoting portal means that the database connection stays safely inside the protected network (assuming the server is secured properly).
