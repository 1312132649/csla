RockfordLhotka replied on Thursday, October 07, 2010

I went through the samples in the CSLA 4 release and removed any that we hadn't had time to update, and that included DeepData. I am not sure a readme would have been terribly helpful since it would just say that the last time DeepData had been updated was perhaps 2 years ago, and that if you want to actually run it you probably have to go back to CSLA .NET 3.0.5 or something.

The Core 3.8 video series includes targeted demos, including the use of the two data access methods I generally recommend (DataPortal_XZY invoking a DAL via repository pattern, or implementing the DAL in an object factory), using EF with DTOs/entities for data transfer to the objects.
